Automations, triggers & hooks

How agents and skills actually run — mapped onto the Agents view, the Workflow canvas, and the Skills panel you've already designed, and grounded in loop-engineering practice. The throughline: triggers wake the loop · the dependency graph decides what's affected · the review rail is where output lands.

1Decouple the three things

Today "an agent" bundles trigger + logic + output. Separate them and the hooks get simple.
Triggers

wake the loop

cron, heartbeat, source-change, webhook, lifecycle, manual. Interchangeable — they only decide when.

→
Dependency graph

decides what's affected

one rule: a write emits a change event; the orchestrator walks the lock-file's reverse edges and marks dependents dirty.

→
Review rail

is where output lands

every result — figure, sentiment, skill flag — arrives as an approve/reject diff with provenance. One grammar.

This is the clean version of "a doc update hooks other docs." No doc-to-doc wiring (hook spaghetti). One event → graph walk → policy decides flag / impact-pass / auto-apply per edge. Scales to millions of edges.

4The Workflow canvas, as a loop

Your canvas is sources → agent → docs. Loop engineering adds two stages: an explicit trigger node at the front, and a verification gate (your Skills as graders) before anything lands — because the writer can't grade its own homework.

The Verify stage is the whole game for trust. Financial agent is a deterministic grader (figures reconcile to sources) — run it first and cheap; Strategy (claims vs OKRs in Knowledge) and Formatting may use a model. If a grader fails, the run stops at the gate and surfaces the flag instead of applying — exactly the maker≠checker split the loop-engineering writing insists on.

6Recommended default automations

A trust-first starter set for the recurring-report beachhead.
⟳

Weekly refresh

cron

Mon 9:00 — re-derive figures, draft commentary, queue meaning-changes for the Monday review.

⚡

Source-change watcher

event

On a source/connector change → walk the graph, flag affected docs. No rewrite — cheap, silent, always-on.

↯

Freshness sweep

heartbeat

Every 6h — impact-pass any flagged-but-unreviewed docs, prepare candidates as drafts for review. Never auto-lands.

⛉

Before-export gate

lifecycle

Financial (must PASS, deterministic) + Formatting run before export; block if figures don't reconcile.

⊚

On-publish snapshot

lifecycle

Pin the doc to current source versions so a published report stays reproducible if sources later move.
